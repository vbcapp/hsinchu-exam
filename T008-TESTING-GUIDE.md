## 📋 測試案例清單

### ✅ 測試案例 1: 取得個人紀錄

**目的**: 驗證 `getUserRecords()` API 能否正確取得用戶紀錄

**步驟**:
1. 點擊「🔍 取得個人紀錄」按鈕
2. 觀察回傳的 JSON 資料
3. 檢查統計卡片是否正確顯示

**預期結果**:
- ✅ API 回應 `success: true`
- ✅ 回傳所有紀錄欄位（`bestDailyAccuracy`, `longestCorrectStreak` 等）
- ✅ 新用戶回傳空白紀錄（所有值為 0 或 null）
- ✅ 統計卡片正確顯示數據

**驗證重點**:
```javascript
{
  success: true,
  data: {
    bestDailyAccuracy: <數字>,
    bestDailyAccuracyDate: <日期 或 null>,
    bestDailyCorrectCount: <數字>,
    longestCorrectStreak: <數字>,
    currentDailyStreak: <數字>,
    // ... 其他欄位
  }
}
```

---

### ✅ 測試案例 2: 更新個人紀錄

**目的**: 驗證 `updateUserRecords()` API 能否正確計算今日統計並更新紀錄

**步驟**:
1. **先確保今天有答題記錄**（可到測驗頁面答幾題）
2. 點擊「🔄 更新個人紀錄」按鈕
3. 觀察「打破紀錄」提示
4. 檢查今日統計數據

**預期結果**:
- ✅ API 回應 `success: true`
- ✅ `brokenRecords` 陣列包含打破的紀錄（若有）
- ✅ `todayStats` 正確顯示今日統計
- ✅ 打破紀錄時顯示金色徽章

**驗證重點**:
```javascript
{
  success: true,
  data: {
    brokenRecords: [          // 本次打破的紀錄
      'bestDailyAccuracy',
      'longestCorrectStreak'
    ],
    currentRecords: {
      todayStats: {           // 今日統計
        accuracy: 92,
        correctCount: 23,
        totalCount: 25,
        avgResponseMs: 3500,
        currentStreak: 8
      },
      // ... 歷史最佳紀錄
    }
  }
}
```

**重要**:
- 若今天尚未答題，`todayStats` 會顯示 `0% / 0 題`
- 若沒有打破任何紀錄，會顯示「今日沒有打破任何紀錄，繼續加油！💪」

---

### ✅ 測試案例 3: 連續更新檢測打破紀錄

**目的**: 驗證不會重複計算同一天的紀錄

**步驟**:
1. 點擊「🏅 測試打破紀錄」按鈕
2. 系統會自動呼叫 `updateUserRecords()` 兩次
3. 觀察兩次的 `brokenRecords` 差異

**預期結果**:
- ✅ 第一次更新：可能打破多項紀錄
- ✅ 第二次更新：`brokenRecords` 應為空陣列（沒有新答題）
- ✅ 顯示「✅ 正確：沒有重複打破紀錄」訊息

**驗證重點**:
```
第一次更新：
打破紀錄數量: 2
打破的紀錄: bestDailyAccuracy, longestCorrectStreak

第二次更新（預期無新紀錄）：
打破紀錄數量: 0
✅ 正確：沒有重複打破紀錄
```

---

### ✅ 測試案例 4: 完整流程測試

**目的**: 模擬完整的答題 → 更新紀錄 → 取得紀錄流程

**步驟**:
1. 點擊「🚀 執行完整流程」按鈕
2. 系統會依序執行：
   - 步驟 1: 更新個人紀錄
   - 步驟 2: 取得最新紀錄
3. 觀察統計卡片

**預期結果**:
- ✅ 步驟 1 成功更新紀錄
- ✅ 步驟 2 成功取得紀錄
- ✅ 顯示 6 個統計卡片（單日最高正確率、單日最多答對、最長連續答對、最長連續天數、當前連續天數、最快平均反應）
- ✅ 兩次 API 回傳的數據一致

**驗證重點**:
- 統計卡片顯示正確（數值、日期）
- 若有打破紀錄，卡片會顯示對應的最新數據

## 🐛 除錯指南

### 問題 1: API 回傳 `success: false`

**可能原因**:
- 未登入
- 資料庫權限問題（RLS 政策）
- `user_records` 表不存在

**解決方法**:
1. 檢查瀏覽器 Console 錯誤訊息
2. 確認已登入（檢查 `currentUserId` 顯示）
3. 確認 T001 SQL 腳本已執行（`user_records` 表已建立）

---

### 問題 2: 今日統計顯示 0

**可能原因**:
- 今天尚未答題
- 時區問題（UTC vs 本地時間）

**解決方法**:
1. 先到測驗頁面答幾題
2. 重新執行測試
3. 檢查 `answer_records` 表是否有今日記錄

---

### 問題 3: 打破紀錄沒有顯示

**可能原因**:
- 今日表現未超越歷史紀錄
- 歷史紀錄已經很高

**解決方法**:
- 這是正常情況！只有真正打破紀錄時才會顯示
- 可嘗試提升今日正確率（答對更多題）

---

### 模擬高正確率（打破紀錄）

若想測試「打破單日最高正確率」，可執行以下步驟：

1. 到測驗頁面，連續答對多題（不中斷）
2. 執行測試案例 2
3. 應會看到 `'longestCorrectStreak'` 在 `brokenRecords` 中
